# KPI definitions

Baseline period 1 started with the 2026-07-15/16 release evidence. Targets remain intentionally unset until two to four weeks of measurements are available.

| KPI | Source | Cadence |
|---|---|---|
| P0 path coverage | risk manifest and security test evidence | weekly |
| Escaped P0/P1 defects | incident records | weekly |
| MTTD / MTTR | alert and incident timestamps | weekly |
| CI pass, runtime, flaky expiry | GitHub Actions artifacts | weekly |
| Audit failures and denial spikes | sanitized telemetry | daily |
| Performance errors/saturation | warning register and monitoring | weekly |
| Change failure rate | release checklist and incidents | monthly |

Targets are set only after two to four weeks of baseline data.
